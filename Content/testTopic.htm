Ticket Types and FanSight Offers

Last week’s release impacts a few offer configuration steps that you need to be aware of.

Ticket Type Option

Ticket Types are now triggered by the Ticket Type field in your offer. You must select the corresponding Ticket Type when creating AXS Premium, ADA, or VIP offers.

Until now, you have had to use very specific status codes in order todisplay the “ticket type” categories in a co-mingled offer. The latest enhancements now make it possible for you to select the Ticket Type in the offer configuration.. Select ticket types during office creation

Every offer without one of these Ticket Types will be considered a “Standard” Ticket Type offer.

About Existing Offers

You may need to update existing Premium | ADA | VIP offers depending on your offer configuration.

  • If the Label in your offer was blank or set to Standard, will use the Status Code (the old method of identifying the Ticket Type).
  • If your offers were set to anything else, you need to add the correct Ticket Type in the offer so they display properly.
  • Please check your Premium and other co-mingled offers now and update them if needed. Do not use the bulk offer edit tool to fix orders for multiple events at once. Doing so will likely wipe out or overwrite your offer images and re-set your offer start and end dates. It is best to update and save each offer manually

Offer Descriptions in Co-mingled Offers

Co-mingled ADA and VIP offers now show the first 40 characters of the offer description for each related offer and a link to the related description when seats are carted.
